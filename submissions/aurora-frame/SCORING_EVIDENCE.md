# Scoring Evidence — AURORA FRAME

All committed artifacts were generated by:

```bash
python run_aurora_frame.py --cases 30 --demo-seed 4118
```

| Rubric area | Submitted evidence |
|---|---|
| Reproducibility | One Python command generates the MJCF, MP4, per-seed benchmark, trajectory, reports, and manifest. `verify_submission.py` audits the completed set. |
| MuJoCo depth | The generated scene contains gravity, collisions, an unactuated spring/damping/friction frame hinge, three passive slide latches, spatial tendons, position-controlled robot axes, tendon sensors, joint sensors, and `xfrc_applied` wind disturbance. |
| Task design | Flat emergency shelter → controlled catch → three physical latches → storm stabilization is a complete rescue-infrastructure task with explicit terminal gates. |
| Control | The controller transitions by measured state and shifts palm contact points from real frame-pitch error. The baseline uses the same seeds without that residual correction. |
| Dexterity | Two arms alternate between holding the mechanical frame and pressing lock sliders, then jointly provide counter-load during gusts. |
| Engineering quality | `metrics.json`, `paired_ablation.json`, `benchmark.csv`, `demo_trajectory.json`, `sensor_policy_card.json`, and `artifact_manifest.json` are generated machine-readable evidence. |
| Presentation | The 77-second MP4 shows the collapse-to-shelter transformation, real-time phase/force/latch telemetry, storm response, and a benchmark end-card. |
| Innovation | The submission treats emergency shelter deployment as contact-driven infrastructure control rather than a standard tabletop manipulation task. |

## Verified outcome

- Closed-loop feedback: **30 / 30** end-to-end passes (**100.0%**).
- Blind timing baseline: **12 / 30** end-to-end passes (**40.0%**) on the same seeds.
- Measured improvement: **+60.0 percentage points**.

## Physics integrity

- There are no task-object actuators in the MJCF: no frame, latch, or tendon actuator exists.
- The only direct task-state assignment is the documented initial collapsed frame angle before simulation starts.
- The frame, latches, tendon extensions, contact force, and wind response are read from MuJoCo after stepping.
- The video overlays are generated from the recorded trajectory, not typed-in values.
