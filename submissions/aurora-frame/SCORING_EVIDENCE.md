# Scoring Evidence - AURORA FRAME 2.0

All committed artifacts are generated by:

```bash
python run_aurora_frame.py --cases 128 --demo-seed 4118 --demo-scenario microburst_recovery
python verify_submission.py
```

| Rubric area | Submitted evidence |
|---|---|
| Reproducibility | One Python command generates the MJCF, MP4, per-seed benchmark, trajectory, reports, and manifest. `verify_submission.py` audits the completed set. |
| MuJoCo depth | The generated scene contains gravity, collisions, an unactuated spring/damping/friction frame hinge, three passive slide latches, spatial tendons, position-controlled robot axes, tendon sensors, joint sensors, contact-force reads, and `xfrc_applied` wind disturbance. |
| Task design | Collapsed emergency shelter -> controlled catch -> three physical latches -> wind stabilization -> adaptive microburst recovery is a full emergency-infrastructure task with explicit terminal gates. |
| Control | The controller transitions by measured state and enters `ADAPTIVE_GUST_RECOVERY` when frame pitch error or microburst load is detected. |
| Dexterity | Two arms alternate between holding the mechanical frame and pressing lock sliders, then jointly provide counter-load while monitoring passive guy-line tendon lengths. |
| Engineering quality | `metrics.json`, `paired_ablation.json`, `benchmark.csv`, `demo_trajectory.json`, `sensor_policy_card.json`, and `artifact_manifest.json` are generated machine-readable evidence. |
| Presentation | The MP4 overlays phase, wind profile, wind direction, frame error, latch state, guy-line load, contact force, gust deflection, and recovery count. |
| Innovation | The submission treats emergency shelter deployment as adaptive infrastructure recovery rather than a standard tabletop manipulation task. |

## Verified outcome

The exact pass counts are generated in `outputs/metrics.json`. The upgraded benchmark contains:

- **128 paired wind trials** for the adaptive policy.
- **128 identical paired wind trials** for the blind-timing baseline.
- Four seeded wind profiles: calm, crosswind, rotating gust, and microburst recovery.
- Recorded recovery events, frame stability margin, contact force, wind profile, and gust response.

## Physics integrity

- There are no task-object actuators in the MJCF: no frame, latch, or tendon actuator exists.
- The only direct task-state assignment is the documented initial collapsed frame angle before simulation starts.
- The frame, latches, tendon extensions, contact force, and wind response are read from MuJoCo after stepping.
- The video overlays are generated from the recorded trajectory, not typed-in values.
